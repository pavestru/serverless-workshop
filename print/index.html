What is Serverless?

  • Compute service to execute code on demand
  • Dynamically allocating the resources
  • Single-purpose, stateless functions
  • Push-based, event-driven pipelines
  • More powerful front ends
  • Embracing third-party services

Why Serverless?

  • Low maintenance
  • Low cost
  • Easy to scale
  • Multiple languages
  • Parallelism
  • Operational management
  • Greener computing
  • Reduced packaging and deployment complexity

  • Develop, deploy, test, secure and monitor

handler.js:

export const hello = async (event, context) => {
  // Do whatever here...
  return {
    statusCode: 200,
    body: JSON.stringify({
      message: `Go Serverless!`
    })
  };
};

serverless.yml:

functions:
  create:
    handler: handler.hello
    events:
      - http:
          path: hello
          method: get

serverless deploy

or

sls deploy
